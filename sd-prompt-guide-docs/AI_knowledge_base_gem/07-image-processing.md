# HiRes 高解析度放大

## 什麼是 HiRes？

**HiRes** （High Resolution Diffusion）是一種「先生成小圖，再放大重繪」的技術，讓你獲得更高解析度、更精緻的圖片。

### 為什麼需要 HiRes？

直接生成大圖會有問題：

| 直接生成大圖          | 使用 HiRes |
|-----------------|----------|
| 容易出現重複元素（多頭、多手） | 構圖正常     |
| 細節可能糊掉          | 細節更精緻    |
| 消耗大量算力          | 效率較高     |

### HiRes 的運作原理
```
步驟 1：生成小圖（如 768×768）
    ↓
步驟 2：將小圖放大（如 1.5 倍 → 1152×1152）
    ↓
步驟 3：對放大後的圖進行「重繪」，補上細節
    ↓
完成：獲得高解析度成品

```

這個「重繪」的過程會參考原本的構圖，同時 AI 會補上放大後缺失的細節。

---

## 關鍵參數：Denoising Strength（去噪強度）

**Denoising Strength** （去噪強度）是 HiRes 最重要的參數，決定了「重繪」時改變原圖的程度。

| 數值            | 效果                   | 適用情境    |
|---------------|----------------------|---------|
| **0.1 ~ 0.3** | 幾乎不改變，只是單純放大         | 只想提升解析度 |
| **0.4 ~ 0.6** | ⭐ **推薦範圍**：補細節但保留構圖 | 大多數情況   |
| **0.7 ~ 0.8** | 會有明顯改變，可能調整細節        | 想要一些變化  |
| **0.9 ~ 1.0** | 幾乎重新畫一張，構圖可能大變       | 基本上不建議  |

### ⚠️ 重要警告

**Denoising Strength 設太高（ > 0.7）容易讓圖片「走樣」！**

常見的走樣情況：

- 人物表情改變
- 髮型或髮色跑掉
- 服裝細節變化
- 背景元素消失或增加
- 整體構圖偏移

**建議**：從 **0.4 ~ 0.5**開始測試，覺得細節不夠再慢慢往上調。

## PixAI 上的 HiRes 設定

### 基本設定流程

1. 生成一張滿意的基礎圖
2. 開啟 HiRes 選項
3. 設定放大倍率（通常 1.5x ~ 2x）
4. 調整 Denoising Strength（建議 0.4 ~ 0.6）
5. 重新生成

### 推薦設定組合

| 目標       | 放大倍率 | Denoising | 說明          |
|----------|------|-----------|-------------|
| 單純放大     | 1.5x | 0.3       | 保持原樣，只提升解析度 |
| 標準 HiRes | 1.5x | 0.5       | 平衡細節與穩定性    |
| 精修細節     | 2x   | 0.5       | 大幅提升解析度     |
| 風格強化     | 1.5x | 0.6       | 讓風格更鮮明      |

## HiRes 與其他功能的搭配

### HiRes + ControlNet

如果你用 ControlNet 控制構圖，HiRes 時也會參考：
```
原圖（有 ControlNet 約束）
    ↓ HiRes 放大
放大圖（繼續維持 ControlNet 的約束）

```

這樣可以在放大時保持姿勢、構圖的一致性。

### HiRes + LoRA

HiRes 階段仍然會套用 LoRA 效果，所以：

- 角色 LoRA 的特徵會被保留
- 風格 LoRA 的畫風會更明顯
- 加速 LoRA 可以加快 HiRes 的重繪速度

## 常見問題

**Q：HiRes 後人物變了怎麼辦？**

A：Denoising Strength 太高了，降到 0.4 以下試試。

**Q：HiRes 後細節反而變糊？**

A：可能是 Denoising 太低（< 0.3），AI 沒有足夠空間補細節。

**Q：HiRes 要跑很久怎麼辦？**

A：可以搭配加速 LoRA（如 LCM）減少步數，或降低放大倍率。

**Q：什麼時候不需要 HiRes？**

A：

- 只是快速預覽構圖時
- 圖片用途不需要高解析度（如縮圖、社群貼文）
- 使用加速 LoRA 快速出圖時

## 進階：Upscaler 的選擇

有些平台讓你選擇不同的 Upscaler（放大演算法）：

| Upscaler                 | 特點        |
|--------------------------|-----------|
| **Latent**               | 標準選項，效果平衡 |
| **Latent (nearest)**     | 保留更多原始細節  |
| **R-ESRGAN 4x+**         | 適合動漫風格    |
| **R-ESRGAN 4x+ Anime6B** | 專為動漫優化    |

PixAI 通常會自動選擇合適的 Upscaler，一般不需要手動調整。

---

## 小結

| 要點                 | 建議                      |
|--------------------|-------------------------|
| Denoising Strength | **0.4 ~ 0.6** （別超過 0.7） |
| 放大倍率               | 1.5x ~ 2x               |
| 使用時機               | 構圖滿意後再放大                |
| 搭配 ControlNet      | 可以維持構圖一致                |

# Inpaint 與 Outpaint 局部編輯

>
> 🎬 **官方教學影片**：[PixAI Inpaint/Outpaint Tutorial](<https://www.youtube.com/watch?v=T7thxTmZEOc>)

---

## 概述

| 功能           | 用途   | 白話說明                 |
|--------------|------|----------------------|
| **Inpaint**  | 局部重繪 | 「把這個區域擦掉，重新畫」        |
| **Outpaint** | 向外擴展 | 「把畫布變大，AI 幫你補上延伸的部分」 |

這兩個功能都是在既有圖片的基礎上，讓 AI「部分重新生成」。

---

## Inpaint 局部重繪

### 什麼是 Inpaint？

Inpaint 讓你用「遮罩」（mask）標記想要修改的區域，AI 會只重畫那個部分，其他地方保持不變。

### 常見應用場景

| 場景             | 說明                         |
|----------------|----------------------------|
| **🖐️ 修復手部崩壞**  | AI 畫的手常常有問題，用 Inpaint 局部重畫 |
| **👁️ 修復臉部問題**  | 眼睛歪斜、表情怪異時局部修正             |
| **🚫 移除不想要的元素** | 背景多出的人、奇怪的物件               |
| **👗 更換服裝**     | 只重畫服裝區域，換一套衣服              |
| **🎨 修改配色**     | 只改頭髮顏色、眼睛顏色等               |
| **➕ 添加元素**     | 在空白處加上飾品、道具                |

### 使用步驟
```
1. 上傳或選擇要修改的圖片
2. 用筆刷標記要重繪的區域（遮罩）
3. 輸入 prompt 描述你想要的結果
4. 調整 Feathering 參數
5. 點擊生成

```

### Feathering 參數

Feathering 控制遮罩邊緣的「羽化程度」，影響重繪區域與周圍的融合效果：

| 數值     | 效果        | 適用情境          |
|--------|-----------|---------------|
| **較低** | 邊緣清晰，融合較少 | 需要精確邊界的修改     |
| **較高** | 邊緣柔和，融合自然 | 希望重繪區域和周圍無縫接合 |

💡 一般建議使用預設值即可。如果重繪區域邊緣有明顯接縫，可以嘗試調高 Feathering。

### Prompt 撰寫技巧

PixAI 的 Inpaint 使用的是PixAI以 Danbooru 為訓練集去訓練的模型（Hikari），因此 **prompt 建議使用 Danbooru tag**。

此外，有一個重要原則：**盡量用「正面描述」取代「否定描述」**。

| 情境         | ❌ 避免        | ✅ 建議                     |
|------------|-------------|--------------------------|
| 想移除人物，補上地板 | `no human`  | `stone floor`            |
| 想移除人物，補上街景 | `no people` | `empty street`、`scenery` |
| 想移除物件      | `no table`  | `wooden floor`、`grass`   |

💡 直接描述你「想要什麼」比描述「不要什麼」更有效。否定詞有時反而會讓模型更容易畫出你不想要的東西。

### 修手小技巧

手部崩壞是 AI 繪圖最常見的問題，Inpaint 修手流程：
```
1. 把崩壞的手部區域塗滿遮罩（畫大一點，包含手腕）
2. Prompt 加上手部描述，例如：
   - "beautiful detailed hands"
   - "correct hand anatomy"
   - "five fingers"
3. 多生成幾次，挑最好的

```

## Outpaint 向外擴展

### 什麼是 Outpaint？

Outpaint 讓你「擴大畫布」，AI 會根據現有圖片的內容，自動補上延伸的部分。

### 常見應用場景

| 場景           | 說明                 |
|--------------|--------------------|
| **📐 改變構圖比例** | 把正方形圖變成橫幅或直幅       |
| **🖼️ 擴展背景**  | 讓風景延伸、增加環境空間       |
| **👤 顯示更多身體** | 半身像擴展成全身像          |
| **📱 適配不同尺寸** | 把圖片調整成適合手機/電腦桌布的比例 |

### 使用步驟
```
1. 上傳或選擇要擴展的圖片
2. 選擇擴展方向（上/下/左/右）
3. 設定擴展的大小
4. 輸入 prompt 描述延伸區域的內容（可選）
5. 點擊生成

```

### Outpaint 注意事項

- **風格一致性**：AI 會嘗試維持原圖的風格，但不一定 100% 完美
- **邊緣接合**：有時候接合處會有明顯的痕跡，可以用 Inpaint 修補
- **多次嘗試**：Outpaint 的結果變化比較大，多生成幾次挑選

---

## Inpaint vs Outpaint 比較

| 項目       | Inpaint | Outpaint  |
|----------|---------|-----------|
| **作用範圍** | 圖片內部    | 圖片外部      |
| **畫布大小** | 不變      | 變大        |
| **主要用途** | 修復、替換   | 擴展、延伸     |
| **需要遮罩** | 是       | 否（選擇方向即可） |

## 實戰範例

### 範例 1：修復六指手
```
問題：AI 畫出了六根手指
解法：
1. Inpaint 遮罩塗滿整隻手
2. Prompt: "1girl, beautiful hand, five fingers, detailed fingers"
3. Negative: "extra fingers, mutated hands, bad anatomy"
4. 生成 4-8 張，挑最好的

```

### 範例 2：移除背景路人
```
問題：背景出現不想要的人物
解法：
1. Inpaint 遮罩塗滿那個人物
2. Prompt: 描述你想要的背景（如 "empty street, no people"）
3. 生成並檢查

```

### 範例 3：把半身像變全身像
```
問題：只有上半身，想看到全身
解法：
1. Outpaint 向下擴展
2. Prompt: "full body, standing, legs, feet, [服裝描述]"
3. 可能需要多次嘗試
4. 接合處不自然的話，用 Inpaint 修補

```

## 常見問題

**Q：Inpaint 後遮罩區域和周圍格格不入？**

A：

- 調高 Feathering，讓邊緣融合更自然
- 遮罩畫大一點，包含一些周圍區域
- 在 prompt 中描述周圍的特徵

**Q：Outpaint 的延伸部分風格不一致？**

A：

- 在 prompt 中強調原圖的風格特徵
- 使用相同的模型和 LoRA 設定
- 嘗試不同的 seed

**Q：修手總是修不好？**

A：

- 遮罩要畫夠大，包含整個手掌和手腕
- Prompt 要明確描述手的狀態（握拳、張開、比 V 等）
- 試試加上 "realistic hands", "anatomically correct"
- 有時候用 ControlNet 的 depth 或 openpose 可以幫助
