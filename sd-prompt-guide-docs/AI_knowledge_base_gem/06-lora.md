## 什麼是 LoRA？

**LoRA** （Low-Rank Adaptation）是一種「微調」技術，讓你可以在不重新訓練整個大模型的情況下，教會模型畫出特定的角色、風格或概念。

### 白話解釋

想像基礎模型是一個會畫畫的人，LoRA 就像是給他的「小抄」：

- **角色 LoRA**：「這個角色長這樣，髮色、眼睛、服裝要這樣畫」
- **風格 LoRA**：「用這種筆觸、配色來畫」
- **概念 LoRA**：「這個姿勢/場景/物件長這樣」

LoRA 檔案通常只有幾十到幾百 MB，比完整模型（幾 GB）小很多，方便下載和切換。

---

## LoRA 的權重概念

權重（Weight）決定 LoRA 對生成結果的影響程度：

| 權重值      | 效果                     |
|----------|------------------------|
| **0**    | LoRA 完全關閉，沒有任何影響       |
| **0.5**  | 效果減半，特徵較淡              |
| **1.0**  | 完整效果（預設值）              |
| **1.2+** | 加強效果，特徵更明顯             |
| **1.5+** | 過度強化，可能出現過擬合（臉崩、顏色過飽和） |

### 實際例子

假設你用一個「藍髮少女」的角色 LoRA：
```
權重 0.3 → 頭髮帶點藍色，但不太明顯
權重 0.7 → 藍髮特徵出現，但可能混合其他顏色
權重 1.0 → 標準的藍髮效果
權重 1.5 → 藍到發亮，可能連皮膚都染上藍色

```

**建議**：先從 1.0 開始測試，如果效果太強就往下調，太弱就往上加。

## LoRA 的種類

在 PixAI 和 CivitAI 上常見的 LoRA 類型：

| 類型          | 用途      | 例子                    |
|-------------|---------|-----------------------|
| **角色 LoRA** | 畫出特定角色  | 雷姆、2B、原神角色等           |
| **風格 LoRA** | 模仿特定畫風  | 厚塗風、水彩風、特定繪師風格        |
| **服裝 LoRA** | 特定服裝設計  | JK 制服、婚紗、盔甲等          |
| **姿勢 LoRA** | 特定動作或構圖 | 戰鬥姿勢、躺姿等              |
| **概念 LoRA** | 特定概念或效果 | 發光效果、特殊背景等            |
| **加速 LoRA** | 減少生成步數  | LCM、DMD2、PCM、Hyper-SD |

## LoRA 疊加技巧

你可以同時使用多個 LoRA！這是 PixAI 上很常見的玩法。

### 常見疊加組合

**組合 1：角色 + 風格**
```
角色 LoRA（權重 0.8）+ 風格 LoRA（權重 0.6）
→ 用特定畫風畫出特定角色

```

**組合 2：角色 + 服裝**
```
角色 LoRA（權重 1.0）+ 服裝 LoRA（權重 0.7）
→ 讓角色穿上不同的衣服

```

**組合 3：角色 + 風格 + 加速**
```
角色 LoRA（權重 0.8）+ 風格 LoRA（權重 0.5）+ LCM LoRA（權重 1.0）
→ 快速生成特定風格的角色圖

```

### 疊加注意事項

1. **總權重控制**：多個 LoRA 疊加時，建議每個都稍微降低權重，避免互相干擾
2. **數量限制**：一般建議 3-5 個以內，太多會讓結果混亂
3. **相容性**：同類型的 LoRA（如兩個角色 LoRA）疊加容易「打架」
4. **加速 LoRA 特殊**：加速 LoRA 一次只用一種，不要混用不同加速方案

### 權重調配範例

| LoRA 組合   | 建議權重                    |
|-----------|-------------------------|
| 單一角色 LoRA | 0.8 ~ 1.0               |
| 角色 + 風格   | 角色 0.7~0.9 / 風格 0.5~0.7 |
| 角色 + 服裝   | 角色 0.8~1.0 / 服裝 0.6~0.8 |
| 三個以上疊加    | 每個都降到 0.5~0.7           |

## 在 PixAI 上使用 LoRA

### 步驟

1. 在生成頁面找到「LoRA」區塊
2. 搜尋或瀏覽想要的 LoRA
3. 點擊加入，調整權重
4. 可重複加入多個 LoRA

### 小技巧

- **看 LoRA 說明**：很多 LoRA 作者會提供建議權重和觸發詞（trigger word）
- **觸發詞**：有些 LoRA 需要在 prompt 中加入特定詞彙才能啟動效果
- **基礎模型相容**：LoRA 是針對特定基礎模型訓練的。例如用 Illustrious XL 訓練的 LoRA，用在 Pony 家族模型上可能效果不如預期

---

## 常見問題

**Q：LoRA 效果不明顯怎麼辦？**

A：檢查以下幾點：

1. 權重是否太低？試著調高到 1.0 或以上
2. 有沒有加觸發詞？查看 LoRA 說明
3. 基礎模型對不對？用 Illustrious XL 訓練的 LoRA 用在 Pony 模型上效果可能會打折

**Q：LoRA 讓圖片崩壞了怎麼辦？**

A：通常是權重太高，試著： 1\. 降低權重到 0.6~0.8 2\. 減少同時使用的 LoRA 數量 3\. 確認 LoRA 和基礎模型相容

**Q：在哪裡找 LoRA？**

A：常見來源：

- PixAI 內建的 LoRA 庫
- [CivitAI](<https://civitai.com>)（最大的社群資源庫）
- [Hugging Face](<https://huggingface.co>)

# 在 PixAI 上訓練自己的 LoRA

> 學會使用 LoRA 之後，下一步就是訓練專屬於自己的 LoRA！PixAI 提供了免費的線上訓練功能，讓你不需要自己準備 GPU 也能輕鬆上手。

---

## 準備訓練圖片

訓練圖片的品質直接影響 LoRA 的效果，**重質不重量**是最重要的原則。

### 基本要求

- **數量**：建議 25～100 張（太少學不好，太多可能過擬合）
- **尺寸**：裁成正方形，邊長至少 768px
- **品質**：寧可高品質的圖片少一點，也不要一堆無關的雜圖

### 角色 LoRA 的特別注意事項

如果你要訓練的是角色 LoRA，還需要注意：

- **臉部清晰置中**：讓模型能好好學習角色的臉部特徵
- **避免遮擋**：盡量不要被手或其他東西遮擋臉部
  - 除非那是你想要訓練的特徵之一（例如角色招牌的手勢）
- **多角度**：最好每個角度都要至少 3 張
  - 正面、側面、45 度角、仰角、俯角

**常見錯誤**：只放正面照 → 模型只會畫正面，換角度就崩
**解決方法**：多準備不同角度的圖片，讓模型學會角色的立體結構

---

## 在 PixAI 上建立訓練

### 步驟 1：進入訓練頁面

在 PixAI 網站上找到「訓練」或「Train」功能（通常在側邊欄或個人頁面中）。

### 步驟 2：填寫基本資訊

| 欄位          | 說明                   | 建議                                             |
|-------------|----------------------|------------------------------------------------|
| **LoRA 名稱** | 你的 LoRA 顯示名稱         | 建議後綴訓練模型，方便辨識（例如：MyCharacter_illustriousXL）    |
| **模型類別**    | 選擇 SD 1.5、SDXL 或 DiT | 目前網站上最多的是 **SDXL**，**DiT**則是未來趨勢，建議至少訓練這兩個版本 |
| **模型主題**    | 選擇訓練用的基礎模型           | 選擇你預計要使用的模型，或至少是同家族的模型                         |
| **觸發詞**     | 啟動 LoRA 效果的關鍵字       | 輸入所有訓練圖片的**共通點**                               |

### 關於觸發詞

觸發詞是讓模型知道「現在要用這個 LoRA」的暗號。設定好的觸發詞應該是：

- 所有訓練圖片都有的共通特徵
- 不會跟其他常見詞彙混淆
- 好記、好打

**範例**：
```
角色名：hatsune miku, aqua hair, twin tails
風格：thick_lineart, painterly
服裝：frilly_maid_dress
```

### 步驟 3：上傳訓練圖片

把準備好的圖片上傳，PixAI 會自動處理後續的訓練流程。

---

## 訓練多個版本

PixAI 現在支援同一個 LoRA 訓練多個版本，這在以下情況很實用：

- 想要針對不同基礎模型各訓練一版（例如 Illustrious XL 和 NoobAI XL）
- 想要訓練適用不同技術模型的LoRA（例如 SDXL 和 DiT）
- 想要更新訓練資料後重新訓練

### 如何新增版本

1. 進入已完成訓練的模型頁面
2. 點擊「**新增版本**」按鈕
3. 或按右上角的選單，選擇「**重複使用資料集**」
4. 選擇新的基礎模型或調整參數
5. 開始訓練新版本

---

## 訓練後的測試

訓練完成後，記得要測試一下效果：

### 測試清單

- **基本測試**：用觸發詞生成，看角色/風格是否正確
- **角度測試**：測試不同視角（正面、側面、背面）
- **權重測試**：測試不同權重（0.5、0.8、1.0、1.2）的效果
- **組合測試**：跟其他 LoRA 疊加看會不會打架

### 如果效果不好

| 問題           | 可能原因       | 解決方法           |
|--------------|------------|----------------|
| 角色特徵不明顯      | 訓練圖片太少或太雜  | 精選更有代表性的圖片重新訓練 |
| 只能畫某個角度      | 訓練圖片角度太單一  | 補充不同角度的圖片      |
| 過擬合（所有圖都長一樣） | 訓練圖片太少或太相似 | 增加圖片多樣性        |
| 跟基礎模型風格不搭    | 訓練用的模型選錯   | 選擇正確的模型主題重新訓練  |

# 加速 LoRA 介紹

## 什麼是加速 LoRA？

一般的圖像生成需要 20-50 步才能完成，**加速 LoRA**透過「蒸餾」能讓你用 **4-8 步**就得到差不多的結果。

### 白話解釋「蒸餾」

你可能會看到「知識蒸餾（Knowledge Distillation）」這個詞，聽起來很像化學實驗，但其實概念很簡單：

> **讓小模型「抄捷徑」學會大模型的生成技巧**

想像一下：

- **老師模型**：很厲害但跑很慢，需要 50 步才能畫出好圖
- **學生模型**：小而快的模型

傳統訓練是讓學生從零開始學（看大量原始圖片），但「蒸餾」的做法是：

> 讓學生**直接模仿老師的作答過程和答案**，而不是重新學習所有原理

就像學生不用把整本微積分課本讀完，只要老師說「遇到這種題型，答案長這樣」，學生就能用捷徑得到差不多的結果。

### 為什麼需要加速 LoRA？

- **節省時間**：快 3-10 倍
- **節省點數/算力**：步數少 = 消耗少
- **快速預覽**：構圖、配色滿意再用正常步數精修
- **批量生成**：大量出圖時效率更高

## 五種推薦的加速 LoRA

以下是在 PixAI 上推薦使用的加速 LoRA，點擊連結可直接前往使用：

### 1\. LCM（Latent Consistency Model）

🔗 **PixAI 連結**：https://pixai.art/model/1687631572599847061

**原理**：將擴散模型蒸餾成「一致性模型」，讓模型學會「一步到位」生成圖像。

**特色**：

- ⭐ 最普及，社群資源最多
- 設定簡單，新手友善
- PixAI 上最常見

---

### 2\. DMD2（Distribution Matching Distillation 2）

🔗 **PixAI 連結**：https://pixai.art/zh/model/1897811443252033347

**原理**：透過「分佈匹配蒸餾」，學生模型快速學會老師模型的輸出分佈。

**特色**：

- ⭐ 畫質比 LCM 更好
- 寫實風格表現特別出色
- 支援極低步數（1-4 步）

---

### 3\. PCM（Phased Consistency Model）- NormalCFG 版

🔗 **PixAI 連結**：https://pixai.art/zh/model/1810142527271081205

**原理**：LCM 的改良版，將生成軌跡分段處理，減少誤差累積。

**特色**：

- ⭐ CFG 彈性最高（可用 2-9）
- 負面提示詞真的有效
- 有 NormalCFG 和 SmallCFG 兩種版本

---

### 4\. WAI-illustrious-Rectified-4Steps

🔗 **PixAI 連結**：https://pixai.art/zh/model/1857294002524835135

**原理**：基於 Rectified Flow 技術，專為 Illustrious 系列模型優化的 4 步加速 LoRA。

**特色**：

- ⭐ 專為 Illustrious 系列優化
- 固定 4 步，設定簡單
- 與 Illustrious 系模型相容性最佳

---

### 5\. Hyper SDXL

🔗 **PixAI 連結**：https://pixai.art/zh/model/1783737893546128417

**原理**：ByteDance 開發的 Hyper-SD 技術，結合 Trajectory Segmented Consistency Distillation（TSCD）與 Human Feedback Learning，實現極低步數生成。

💡 **小知識**：雖然名稱是「2steps-lora」，技術上確實支援 2 步生成，但 PixAI 上建議使用 8 步以獲得更穩定的畫質。

**特色**：

- ⭐ 建議 8 步，可更高
- CFG 需維持低值（1.1-2.0）
- 採樣器相容性佳（Euler a、DDIM）

---

## 建議參數表

在 PixAI 上使用這些加速 LoRA 時的推薦設定（實際設定值仍需以LoRA介紹頁的建議以及自己需求為主）：

| 項目          | LCM     | DMD2    | PCM (NormalCFG) | WAI-Rectified-4Steps | Hyper SDXL     |
|-------------|---------|---------|-----------------|----------------------|----------------|
| **建議步數**    | 3-8     | 4-8     | 4-16            | 4                    | 8+             |
| **CFG 範圍**  | 1.0-2.0 | 1.0-1.6 | 2-9             | 1.0-2.0              | 1.1-2.0        |
| **採樣器**     | LCM     | LCM     | DDIM / Euler    | Euler                | Euler a / DDIM |
| **LoRA 權重** | 1.0     | 0.7-1.0 | 1.0             | 1.0                  | 1.0            |

### PCM 版本說明

PCM 有兩種版本，差別在於 CFG 範圍：

| 版本            | CFG 範圍 | 適用情境         |
|---------------|--------|--------------|
| **NormalCFG** | 2-9    | 需要負面提示詞發揮作用時 |
| **SmallCFG**  | 1-2    | 類似 LCM 的使用方式 |

## 優缺點比較

### LCM

| 優點 ✓      | 缺點 ✗        |
|-----------|-------------|
| 最普及、社群資源多 | CFG 只能用 1-2 |
| 相容性好      | 負面提示詞效果差    |
| 設定簡單易上手   | 步數變化結果不一致   |

### DMD2

| 優點 ✓        | 缺點 ✗       |
|-------------|------------|
| 畫質比 LCM 更好  | 可能出現泛白/褪色  |
| 支援極低步數（1-4） | 需調整 CFG 補償 |
| 寫實風格表現佳     | 社群資源較少     |

| 優點 ✓         | 缺點 ✗                |

| 負面提示詞有效      | 需選對版本（Normal/Small） |
| 不同步數結果穩定     | 步數要匹配 LoRA 版本       |

### WAI-illustrious-Rectified-4Steps

| 優點 ✓              | 缺點 ✗               |

| 專為 Illustrious 優化 | 僅適用 Illustrious 系列 |

| 相容性佳              | 其他模型效果可能不佳         |

### Hyper SDXL

| 優點 ✓    | 缺點 ✗              |

| 採樣器相容性佳 | CFG 必須很低（1.1-2.0） |
| 設定簡單易上手 | 高 CFG 會出問題        |
| 畫質穩定    | 步數不算最少（8+）        |

## 實用建議

### 1\. 先用加速 LoRA 預覽，再用正常步數精修

加速 LoRA 最適合用來快速確認構圖和配色，滿意後再關掉加速 LoRA，用 20-30 步精修細節。

### 2\. 加速 LoRA 可以和其他 LoRA 疊加

你可以同時使用：

- 加速 LoRA（如 LCM）
- 風格 LoRA
- 角色 LoRA

只要注意總權重不要太高，避免互相干擾。

### 3\. 不同加速 LoRA 不要混用

一次只用一種加速 LoRA，混用會導致結果混亂。

### 4\. 注意版本匹配

- **SDXL 模型**用 SDXL 版的加速 LoRA
- **SD 1.5 模型**用 SD 1.5 版的加速 LoRA
- **Illustrious 系列**優先考慮 WAI-Rectified-4Steps
- 版本不對會完全沒效果或出錯
